1. PostgreSQL sync engine — the biggest gap
The old system has a full db/sync.py — a SyncManager class that does bidirectional push/pull between SQLite and PostgreSQL, with a sync log, schema mirroring, and UUID-based conflict resolution. The new system has the PostgreSQL settings panel and a test connection button, but no actual sync logic at all. Saving the PG credentials does nothing — there's no code that ever pushes or pulls data.
2. Normal A4 printer support for receipts
The old system has printing/normal_printer.py — it sends the receipt text to a standard Windows/Linux printer via QPrinter, using a monospaced font so it looks like a thermal receipt printed at the top of the page. The new system has a normal_printer_name setting stored but print_manager.py never uses it — all printing goes to the thermal path only. So if someone doesn't have a thermal printer and just has a regular office printer, receipts won't print.
3. Group profit margin on the product form
The old system shows a live auto-calculation hint when you enter a cost price — it looks up the group's profit margin and shows the computed selling price inline (e.g. = $5.00 × (1 + 25%) = $6.25). The new system does the calculation but the hint label only appears when the cost or group changes, and the profit margin itself isn't editable from the product form — you have to go to the Groups panel to change it.
4. Those are the three real gaps. Everything else — users, quick keys, DBF import, void/refund, stock, price tags, reports, sessions — is present in the new system and in most cases improved.

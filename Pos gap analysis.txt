# POS Migration — Gap Analysis
**Old project:** `pos-system` (Python/PyQt6, monolithic UI files)
**New project:** `merchant_pos copy` (Python/PyQt6, modular structure)

## ❌ What's Missing from the New Project

These features exist in the old project but have **not yet been ported** to the new one.

### 1. 🖨️ Thermal Printer Driver
**Old:** `printing/thermal_printer.py` — full
**New:** `utils/print_manager.py` only handles standard system printers (QPrinter)

Missing:
- `auto_detect_printer()` — USB → serial → network auto-detection
- `_try_usb(vendor_id, product_id)` — USB connection (Epson VID default)
- `_try_serial(port)` — serial/RS-232 connection
- `_try_network(host)` — TCP/IP connection
- `print_text_thermal(text, cut=True)` — ESC/POS send + paper cut
- Thermal connection type stored in settings (`thermal_connection`, `thermal_port`, `thermal_vendor_id`, `thermal_product_id`)

**Action:** Port `printing/thermal_printer.py` → `utils/thermal_printer.py`

---

### 2. 🧾 Receipt Formatter (Thermal Layout)
**Old:** `printing/formatter.py` — formats receipt text for 42-char thermal rolls
**New:** `utils/print_manager.py` has a basic `_format_receipt_text()` but it lacks:

Missing:
- `format_void(data, width)` — void notice receipt
- `format_refund(data, width)` — refund receipt
- `format_session(data, width)` — session summary printout
- `_format_items_gct_column(items, width)` — 42-char layout with GCT column per item
- `_format_items_simple(items, width)` — simple layout without GCT breakdown
- Receipt layout choice (`gct_column` vs `simple`) stored in settings

**Action:** Port `printing/formatter.py` → `utils/receipt_formatter.py`

---

### 3. 🧾 Receipt Builder (DB → Print Data)
**Old:** `printing/receipt_builder.py` — fetches all data from DB and builds print dicts
**New:** `utils/print_manager.py` does this inline but is missing:

Missing:
- `build_void_receipt(transaction_id, reason, voided_by)` — void notice data
- `build_refund_receipt(...)` — refund receipt data
- `build_session_receipt(session_id)` — session summary data

**Action:** Port `printing/receipt_builder.py` → `utils/receipt_builder.py`

---

### 4. 🖨️ Print Manager — Void, Refund, Session
**Old:** `printing/print_manager.py`
**New:** `utils/print_manager.py` only has `print_receipt()` and `print_label()`

Missing functions:
- `reprint_receipt(transaction_id, parent)` — reprint without cash/change
- `print_void(transaction_id, reason, supervisor_name, parent)` — print void notice
- `print_refund(transaction_id, refund_items, refund_total, reason, refunded_by, parent)` — print refund receipt
- `print_session(session_id, parent)` — print session summary
- `_dispatch(text, printer_type, ...)` — route to thermal vs normal

**Action:** Extend `utils/print_manager.py` with these functions

---

### 5. 🏷️ Shelf Label Maker
**Old:** `supervisor_dashboard.py` → Labels tab with full label designer + printer
**New:** `labels/` folder exists but is empty. `utils/print_manager.py` has `print_label()` stub.

Missing:
- Label designer UI (product list with qty selectors, preview panel)
- `_label_load_products(query)` — searchable product list for label selection
- `_label_select_all() / _label_clear_selection()`
- `_label_update_preview()` — live label preview widget
- `_label_get_siblings(pid)` — get alias siblings for multi-barcode labels
- `_label_print(save_pdf=False)` — print or save PDF
- `_draw_label(painter, rect, product, options, preview)` — render label with barcode
- `_draw_barcode_bars(painter, rect, barcode_text)` — barcode rendering
- Label options: width/height mm, show price, show barcode, font size

**Action:** Port Labels tab from `supervisor_dashboard.py` → `ui/supervisor/labels_tab.py`

---

### 6. 🔄 PostgreSQL Bidirectional Sync
**Old:** `db/sync.py` — full `SyncManager` class with push/pull/sync
**New:** PostgreSQL config is stored and there's a `_pg_test()` in manager UI, but no actual sync engine.

Missing:
- `SyncManager` class with `push_all()`, `pull_all()`, `sync()`
- `_mirror_schema(pg_conn, table, sqlite_db_path)` — replicates SQLite schema to PG
- `_push_table / _pull_table` — per-table sync
- `sync_log` table in checkout.db for sync history
- `sync_id` UUID and `synced_at` columns on synced tables
- Sync UI panel in manager (progress bar, log output, push/pull/sync buttons)

**Action:** Port `db/sync.py` → `core/db_sync.py` and add Sync tab to manager window

---

### 7. 📦 DBF Stock Import
**Old:** `import_stock_dbf.py` + Manager → DBF tab
**New:** Not present at all.

Missing:
- `import_stock_dbf.py` — reads legacy `.dbf` files and imports to products.db
- Manager DBF tab with file picker, progress bar, import log
- Background thread with progress signals

**Action:** Port `import_stock_dbf.py` and the DBF tab from `manager_dashboard.py`

---

### 8. 🔧 First-Run Setup Wizard
**Old:** `ui/setup_wizard.py` — multi-step wizard shown on first launch
**New:** `init_db()` in `db_users.py` silently seeds a default `manager`/`admin` account.

The old wizard guided the user through:
1. Welcome screen
2. Create manager account (choose username + password)
3. Set business name + GCT rate

**Action:** Either port `ui/setup_wizard.py` or keep the silent seed (acceptable if you document the default credentials).

---

### 9. 📊 Cashing Sessions (Supervisor-Managed Trading Periods)
**Old:** `cashing_sessions` table — supervisor opens/closes a period for each cashier; tracks totals per period
**New:** Only `sessions` table (one per login). No supervisor-managed cashing sessions.

Missing:
- `cashing_sessions` table in checkout.db (or users.db)
- Open/close session buttons in supervisor Reports tab
- Session totals cards (total sales, GCT, discount, transaction count)
- `_rpt_open_session()` / `_rpt_close_session()` properly write to `cashing_sessions`

> **Note:** The new `supervisor_window.py` has `_rpt_open_session()` and `_rpt_close_session()` stubs but they reference the sessions table (login sessions), not a separate cashing_sessions concept. Decide if you want to keep the distinction.

---

### 10. 🔁 Reprint Last Receipt (Cashier Dashboard)
**Old:** `_reprint_last_receipt()` on the cashier dashboard
**New:** `_reprint_last()` exists in `cashier_window.py` but calls `reprint_receipt()` which doesn't exist in `utils/print_manager.py` yet.

**Action:** Implement `reprint_receipt(receipt_id)` in `utils/print_manager.py` (covered under item 4 above).

---

### 11. 💰 Selling Price Auto-Calculation (Cost + Group Profit %)
**Old:** `recalculate_selling_prices()` in `models.py` — full cascade logic
**New:** `_calc_selling_price()` and `_get_selling_price()` exist in `supervisor_window.py` as UI helpers only. There's no DB-level cascade function.

Missing:
- `recalculate_selling_prices(product_ids, group_id, all_products)` — DB function that recalculates and saves
- `recalculate_all_cases(case_profit_pct)` — recalculates all case prices when case profit % changes
- Cascade logic: when a single's cost changes → update its linked case's cost and selling_price

**Action:** Port these functions from `db/models.py` → `core/db_products.py`

## 📋 Recommended Migration Order

Work through these in order — each one unblocks the next:

1. **`core/db_products.py`** — Add `recalculate_selling_prices()` and `recalculate_all_cases()` (needed by supervisor save product)
2. **`utils/thermal_printer.py`** — Port ESC/POS driver from old `printing/thermal_printer.py`
3. **`utils/receipt_formatter.py`** — Port formatter (void, refund, session, GCT-column layouts)
4. **`utils/receipt_builder.py`** — Port receipt/void/refund/session data builders
5. **`utils/print_manager.py`** — Add `reprint_receipt`, `print_void`, `print_refund`, `print_session`, thermal dispatch
6. **Printer settings UI** — Add thermal connection fields to manager Printers panel (already has normal printer fields)
7. **Labels tab** — Port shelf label designer to `ui/supervisor/labels_tab.py`, add to both supervisor and manager
8. **`core/db_sync.py`** — Port `SyncManager` from old `db/sync.py`
9. **Sync UI tab** — Add to manager window (progress bar + push/pull/sync buttons)
10. **DBF import** — Port `import_stock_dbf.py` and DBF tab to manager
11. **Cashing sessions** — Decide if you want supervisor-managed periods; if yes, add `cashing_sessions` table and update Reports tab

## 🗄️ Database Schema Differences

| Concept | Old column/table | New column/table | Action |
|---|---|---|---|
| Alias | `aliases` table (shared across products) | `product_alias` table (per product) | Different model — new is simpler |
| Discount levels | `discount_levels` table (separate rows) | `discount_level1/2` INT columns on product | New is simpler but less flexible |
| Case profit % | `business_info.case_profit_percent` | `settings` key `case_profit_pct` (add it) | Add to `_DEFAULTS` in `db_config.py` |
| Thermal printer settings | columns in `business_info` | `settings` keys (partially there) | Add `thermal_connection`, `thermal_port`, `thermal_vendor_id`, `thermal_product_id` keys to `_DEFAULTS` |
| Receipt layout | `business_info.receipt_layout` | Add `receipt_layout` to settings `_DEFAULTS` | |
| Cashing sessions | `cashing_sessions` table | Not present | Add if needed |
| Sync columns | `sync_id`, `synced_at` on each table | Not present | Add when implementing sync |
